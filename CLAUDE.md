# Desktop Organizer - 프로젝트 규칙

## 문서 작성 지침

### README 작성
- **독자**: 초보자부터 개발자까지 (균형잡힌 설명)
- **톤**: 친근하고 명확하게, 이모지 적절히 사용
- **구조**: 개요 → 주요 기능 → 설치 → 사용법 → 주의사항 → 기여 방법
- **상세도**: 초보자도 따라할 수 있도록 단계별 설명 포함

## 코드 작업 지침

### Plan Mode 사용 기준
다음의 경우 **반드시 Plan Mode**를 사용하여 검토:
- `.gitignore` 파일 수정
- `CATEGORY_MAP` 수정 (위험 확장자 .sys, .dll 등 확인)
- `move_files()` 등 파일 이동 로직 수정
- 새 파일 생성 시 민감 정보 포함 여부 확인

### 안전 규칙
**절대 추가 금지 확장자** (CATEGORY_MAP):
- `.sys`, `.dll`, `.exe`, `.bat`, `.cmd` (시스템 파일)
- `.ini`, `.conf` (설정 파일)

**Git 제외 대상** (.gitignore):
- API 키, 비밀번호 등 민감 정보
- 캐시 파일 (`__pycache__`, `*.pyc`)
- OS 파일 (`.DS_Store`, `Thumbs.db`)

## 커밋 전략

### 커밋 메시지 형식
- 한글로 작성
- 형식: `[유형]: [내용]`
- 예시:
  - `문서: README 사용성 개선`
  - `기능: 통계 출력 기능 추가`
  - `버그: 인코딩 에러 수정`

### 커밋 단위
현재 학습 단계에서는:
- **학습 단계별로 커밋** (Phase별)
- 문서 수정은 관련 내용 묶어서 한 커밋
- 코드 수정은 기능별로 분리
